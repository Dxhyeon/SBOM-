Automated Vulnerability Identification System Using SBOM

SBOM을 이용한 취약점 식별 자동화 시스템

  • 01 SBOM이란 무엇인가요?

    SBOM은 소프트웨어 빌드를 구성하는 모든 구성 요소와 해당 소프트웨어가 생성된 환경을 식별하는 문서입니다.
    공급자의 정보, 각 컴포넌트의 이름, 고유한 식별자, 버전, 컴포넌트의 해시, 종속성 및 관계, 작성자의 정보, 라이선스 정보, 취약성 정보 등이 포함됩니다.

  • 02 어떻게 취약점을 판별하나요?

    CVE, CWE 내용에 근거하여 SBOM과 비교하여 사용된 오픈소스의 취약점 관리 및 발생 위치를 판별함으로 취약점을 진단합니다.

  • 03 관리서비스란 무엇인가요?

    저희 시스템은 대시보드를 통해 취약점 정보를 시각화하여 제공합니다. 뿐만 아니라, 취약점 정보를 XML 파일 혹은 이메일을 통해 신속하게 통보해 드립니다.
    더불어 SBOM의 정보도 함께 제공하여 고객님들이 보다 종합적으로 소프트웨어를 관리하실 수 있습니다.

Frequently Asked Questions(FAQ)

'SBOM을 이용한 취약점 식별 자동화 시스템'에 대해 자주 묻는 질문과 그에 대한 답변을 제공합니다. 자세한 내용은 아래를 참조하세요.

  • 어떻게 SBOM을 생성하고 업로드할 수 있나요?

    SBOM은 일반적으로 소프트웨어 개발 단계에서 자동으로 생성됩니다. 생성된 SBOM을 시스템에 업로드하면 자동으로 취약점을 식별할 수 있습니다.

  • CVE 코드는 무엇을 의미하나요?

    CVE 코드는 '공개적으로 알려진 취약점'을 나타내는 식별자입니다. 이 코드를 통해 특정 취약점을 찾고 관리할 수 있습니다.

  • 이 시스템은 어떤 유형의 소프트웨어에 적용 가능한가요?

    이 시스템은 다양한 종류의 소프트웨어에 적용 가능합니다. 웹 애플리케이션, 모바일 앱, 데스크톱 프로그램 등 다양한 플랫폼에서 사용할 수 있습니다.

  • SBOM을 활용한 취약점 식별은 왜 중요한가요?

    SBOM을 활용하면 소프트웨어의 구성 요소와 취약점을 명확하게 파악할 수 있어 보안을 강화하고 신속한 대응이 가능합니다.

  • 이 시스템을 사용하면 어떤 이점이 있나요?

    이 시스템을 사용하면 소프트웨어의 취약점을 신속하게 식별하여 대응할 수 있습니다. 또한, SBOM 정보를 XML 파일로 다운로드하여 효율적으로 관리할 수 있습니다.
